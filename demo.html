Live Product Story

One inbound message becomes an action plan.

This is the GlowFlow product pitch in one screen: plain-English code receives an inbound message, extracts structured data, classifies the urgency, decides the next step, and drafts a reply.

v1.0.1 Extract structured fields Decide next action Draft a reply
GlowFlow / GlowScript
set inbound_message to "Name: Ana email ana@example.com company Glow. Urgent request. Missing shipping address. Customer asked for pricing."

extract from inbound_message:
  name
  email
  company

classify inbound_message as:
  urgent
  normal
  spam

decide next_step from inbound_message:
  ask_for_details
  send_invoice
  escalate_to_human

rewrite "Thanks for reaching out. We can help right away." in friendly tone
set draft_reply to result

What Glow Is Doing
1. Understand Turns unstructured text into named fields like name, email, and company.
2. Triage Classifies the message and decides what should happen next.
3. Act Triggers workflow behavior like approvals, webhooks, tools, and file actions.
4. Stay Readable Keeps the automation in a script that reads like instructions, not framework code.
Run It
cargo run -- run examples/investor_demo.glow
./scripts/investor-demo.sh
