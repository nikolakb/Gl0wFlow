Pressure Point

AI automation breaks down when the workflow stops being readable.

The pressure point is not writing one API call. It is what happens after the fifth webhook, the third approval step, the second AI classification, and the first urgent production edit. At that point, teams usually end up with Python glue, shell fragments, YAML sprawl, and prompt text scattered across files that no longer read like one coherent system.

  • Logic gets split across code, config, prompts, and workflow tools.
  • AI behavior becomes hard to audit because intent is no longer explicit.
  • Simple automation changes start requiring framework-level edits.
  • Operational pressure grows faster than readability.
Solution

GlowFlow brings the workflow back into one readable language surface.

GlowFlow solves that by making GlowScript itself the control plane for the workflow. Files, webhooks, AI extraction, decisions, tools, approvals, schedules, and API calls all stay visible in one plain-English script instead of being spread across multiple layers.

  • Readable source becomes the operational source of truth.
  • Structured AI operations become language forms, not scattered prompt glue.
  • The same script can be checked, run, served, inspected, and built.
  • The system stays small enough to reason about under change pressure.
Quick Start

Start from the CLI, not from a framework.

cargo run -- run examples/investor_demo.glow
cargo run -- check examples/mother_of_all.glow
cargo run -- build examples/mother_of_all.glow

./scripts/investor-demo.sh

GlowFlow keeps the entry point small. You run a GlowScript file, inspect it, build it, or serve it. The docs below are organized around that same workflow.

Proof

What the current proof surface actually says

The strongest claim GlowFlow can make right now is not marketing language. It is that the language boundary is real and the runtime behavior has survived repeated heavy passes.

Language

9 passing tests

The public suite is intentionally compact: one full end-to-end mother scenario, one narrow runtime regression test, and seven focused v1.0.1 fix tests.

Stress

Heavy churn already found real bugs

Same-file formatter corruption, webhook GET mutation, import-base inconsistencies, and provider fenced-JSON interop were all exposed by stress passes and fixed.

AI Providers

Real Ollama-backed proof

GlowFlow now has verified runs against a local Ollama server for extract, classify, decide, and summarize.
